# NPC History Layer — WorldMonitor Integration Notes

## Concept

Add a historical time slider to the WorldMonitor 3D globe that renders
grid-cell-level "NPC History" data from the `xcu_my_apps/gridded_world_history`
framework. The globe becomes a time machine.

## Proposed New Map Layers

### Population Density NPC Heatmap
Color terrestrial cells by estimated population density per era. Uses HYDE
population grid data (already available in gridded_world_history).

### Life Expectancy Gradient
Estimated lifespan by cell and era. Gradient from red (short) to green (long).

### Information Horizon
For a selected cell, show what percentage of the world it has any awareness of
at a given date. Renders as a fading circle/ellipse from the selected cell,
shaped by trade routes and geography.

### Migration Pressure Vectors
Arrows showing likely movement direction based on anthrome transition pressures
(e.g., desertification pushing people toward river valleys).

### "What Would You Be?"
Click a cell + date → get your probable NPC archetype with a micro-narrative
generated by the LLM pipeline.

## Time Slider Design

- Range: 10000 BCE → present
- Logarithmic scale (more resolution for recent centuries)
- Play/pause animation mode (sweep through history)
- Syncs with existing variant panel system
- When in historical mode, live news panels dim/hide; NPC narrative panel appears

## Data Pipeline

```
gridded_world_history (Python)
    → century_aggregator produces per-cell anthrome data
    → NPC generator (TrillionsOfPeople) creates representative personas
    → JSON/GeoJSON export per century
        → deck.gl GeoJsonLayer on WorldMonitor globe
```

## Implementation Notes

- The `codexes` variant already supports iframe embedding — historical mode
  could be a new variant (`VITE_VARIANT=history`) or a toggle within `full`
- deck.gl GeoJsonLayer can handle 82k cells if we use binary attributes
- Time slider component follows the same pattern as RecencyFilter
- Consider using H3 hexagons (already in gridded_world_history core.py) for
  a more visually appealing hex-grid overlay
